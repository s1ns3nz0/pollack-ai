## 와치리스트란?
Sentinel KQL 룰 안에 하드코딩된 값 대신, **외부에서 동적으로 참조할 수 있는 CSV 기반 목록**.
```kql
기존 방식:
| where Operator in ("lt.kim", "capt.park")  ← 룰 파일 직접 수정 필요

와치리스트 방식:
| where Operator in (_GetWatchlist('Approved_Operators_List') | project Operator)
← 와치리스트 값만 바꾸면 룰은 그대로
```

---

## 왜 필요한가?
에이전트가 KQL 룰(ARM 템플릿)을 직접 수정하려면 GitHub PR → CI/CD 배포 과정이 필요해서 느리고 위험해. 와치리스트는 Sentinel API 한 번 호출로 즉시 반영되기 때문에, **Rule Update Agent가 오탐 학습 결과를 안전하게 반영하는 통로**가 된다.
```
ARM 템플릿(룰 파일) 수정
  → GitHub PR → CI/CD 배포 필요 → 느리고 위험

와치리스트 수정
  사람:    Azure Portal → Sentinel → 위치 목록 → 항목 편집
  에이전트: Sentinel API PATCH 호출로 예외 목록 자동 추가
  → 룰 파일 건드리지 않고 즉시 반영, 롤백 쉬움
```

## 어떤 식으로 쓸 건지 — 에이전트 파이프라인 연동
```
Sentinel Alert 발생
    ↓
Triage Agent
  → Approved_SystemId_List 읽기
    "이 UAVId 인가된 기체?" → 미등록이면 심각도 상향
    ↓
Investigation Agent
  → Approved_Operators_List   "이 Operator 인가된 사람?"
  → Approved_Firmware_Hash_List "이 해시 정상 펌웨어?"
  → AOI_Boundary_List         "이 좌표 허가 구역?"
    ↓
Validation Agent
  → GNSS_Exception_List       오탐 위치 예외 처리
  → C2_Whitelisted_GCS_List   오탐 GCS IP 예외 처리
    ↓
Rule Update Agent
  → 반복 오탐이면 와치리스트에 자동 추가 (학습)
    ↓
Response Agent
  → Approved_Weapon_List      "이 WeaponId 등록된 무기?"
```

## 시나리오 → 와치리스트 매핑
S1~S11은 LIG 해커톤이 정의한 **공격 시나리오 번호**, B-1~B-13은 양진수님 운용개념 문서에서 정의한 **탐지 항목 번호**. 서로 다른 분류 체계지만 대응관계가 있음 (B-6 = S4, B-5 = S3, B-9 = S6).

현재 배포된 룰은 S1~S6 기준.

| 시나리오 | 와치리스트 | 의미 / 정의 | 왜 필요한지 | 상태 |
|---|---|---|---|---|
| **S1** GNSS 스푸핑 | `GNSS_Exception_List` | 스푸핑이 아닌 정상 예외 좌표 목록 | 안흥 기지 주변 GPS 반사파 등 환경적 이상을 오탐 처리 | ✅ 배포완료 |
| **S2** C2 하이재킹 | `C2_Whitelisted_GCS_List` | 인가된 GCS IP/ID 목록 | 정상 GCS도 비인가 IP처럼 보일 수 있음 → 오탐 방지 | ✅ 배포완료 |
| **S4** 펌웨어 공급망 | `Approved_Firmware_Hash_List` | 버전별 정상 펌웨어 해시 목록 | 해시 불일치 탐지 시 "어떤 해시가 정상인지" 기준 필요 | ❌ 진수님 해시값 필요 |
| **S4** 펌웨어 공급망 | `SBOM_Approved_Package_List` | 펌웨어에 허가된 패키지 화이트리스트 | `SbomForbiddenCount` 숫자만으론 어떤 패키지가 미허가인지 모름 | ❌ 진수님 확인 필요 |
| **S5** 방어 회피 | `Approved_SystemId_List` | 인가된 기체 UAVId 목록 | 레거시 기체(MUAV-001 등) 데이터가 섞여서 오탐 발생 | ✅ 배포완료 |
| **S6** GCS 측면 침투 | `Approved_Operators_List` | 승인된 운영자 계정 목록 | 미등록 계정 로그인 = 계정 탈취 탐지 기준 | ✅ 배포완료 |
| **S3** SATCOM MITM | — | 와치리스트 불필요 | `IntegrityStatus != 'ok'` 단독으로 탐지 가능 | ⏸ A-4 데이터 필요 |

> **B섹션 탐지 항목과 와치리스트**
> 양진수님 운용개념 문서(B-1~B-13) 탐지 항목 중 와치리스트가 필요 없는 것들:
> - **B-2** 속도/고도 → "Airspeed > 70m/s" 같은 임계값 기반 탐지
> - **B-3** 미션 조작 → "1분에 5번 이상" 식의 횟수 기반 탐지
> - **B-4** mavlink 무결성 → UnsignedCount/FailedCount 카운터 기반 탐지
> - **B-5** SATCOM → IntegrityStatus 상태값 기반 탐지
> - **B-7** C4I/작전 → 시간대/횟수 기반 탐지 (단, Operator 인가 여부는 `Approved_Operators_List`로 커버 — S6에서 이미 배포)
> - **B-11** 영상 → 콴/AOI 기반 탐지
> - **B-12** Failsafe → 동시 리셋 횟수 기반 탐지
> - **B-13** 편대 패턴 → 거리/표준편차 기반 탐지
>
> B-1(AOI 이탈), B-10(무장 이상)은 와치리스트 필요하지만 지금 S1~S6 범위 밖 → "추가로 고려할 수 있는 것" 섹션 참고.

## 배포 현황

| 와치리스트 | 상태 | 검색키 | 에이전트 자동 쓰기 |
|---|---|---|---|
| `GNSS_Exception_List` | ✅ 배포완료 | Lat | ✅ Rule Update Agent (반복 오탐 위치 자동 등록) |
| `C2_Whitelisted_GCS_List` | ✅ 배포완료 | GCS_IP | ✅ Rule Update Agent (반복 오탐 IP 자동 등록) |
| `Approved_Operators_List` | ✅ 배포완료 | Operator | ❌ 사람만 수정 가능 |
| `Approved_SystemId_List` | ✅ 배포완료 | UAVId | ❌ 사람만 수정 가능 |
| `Approved_Firmware_Hash_List` | ❌ 진수님 해시값 필요 | FirmwareVersion | ❌ 사람만 수정 가능 |
| `SBOM_Approved_Package_List` | ❌ 진수님 확인 필요 | PackageName | ❌ 사람만 수정 가능 |

> **원칙**: 신원/자격 관련(Operators, SystemId, Firmware)은 사람만 수정. 행동 패턴/구역 관련(GCS IP, GNSS 위치)만 에이전트 자동 쓰기 허용.
